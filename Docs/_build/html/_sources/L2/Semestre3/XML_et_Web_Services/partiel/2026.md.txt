# Exercice 2 : Schema XSD (9 pts)

Mireille, une doyenne informaticienne d’une banque de la place, gère le virement  
des salaires des agents des sociétés de la place via les fichiers TXT. Après une mise  
à jour sécuritaire du système informatique de la banque, seuls les fichiers XML  
sont autorisés.

À cet effet, il vous est demandé, vous jeune et dynamique  
informaticien, de mettre en place le schéma XSD validant les documents XML.

## Remarques

- Le sexe : **M** ou **F** **(1 pt)**
- La nationalité : **AMERICAINE, CANADIENNE, CHINOISE, JAPONNAISE** **(1 pt)**
- Nombre de personne à charge : **entre 0 et 10** **(1.5 pt)**
- Le sang : **A+, A-, B+, B-, O+, O-, AB+, AB-** **(1.5 pt)**
- La taille : **entre 1.0m et 3.0m** **(1 pt)**
- La masse : **entre 40kg et 120kg** **(1 pt)**
- À chaque société correspond un fichier XML
- Dans chaque fichier XML, il y a **au moins trois (03) employés** **(1 pt)**
- Schéma XSD bien formé **sans espace de nom** **(1 pt)**

## Extrait de la structure des fichiers XML

```xml

<virement>

    <employee society_id="001">

        <nationalite>CANADIENNE</nationalite>
```

# Exercice 3 : XPATH (2.5 pts)

1. Écrire une expression XPath pour sélectionner tous les noms (`<nom>`) des employés. **(0,5 pt)**

2. Sélectionner les employés qui ont au moins une personne à charge (`<personcharge> > 0`). **(0,5 pt)**

3. Écrire une requête XPath pour sélectionner l’avant-dernier employé de la liste des employés. **(0,5 pt)**

4. Sélectionner les employés ayant le sexe féminin (`<sexe> = 'F'`) et le groupe sanguin est `O+`. **(1 pt)**

5. Sélectionner les employés dont la taille est supérieure à `1.85`. **(0,5 pt)**
